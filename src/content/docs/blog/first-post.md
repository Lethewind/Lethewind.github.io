# 我的第一篇文章

这里是正文开头。你可以先用 2~3 句话说明这篇文章想回答什么问题、写给谁看，以及为什么值得记录下来。

## 背景

- 这篇文章的起因是什么？
- 当时在做什么项目或遇到了什么问题？
- 为什么最后决定把它写成文档？

## 我做了什么

可以按时间线或步骤来写：

1. 第一步做了什么。
2. 第二步遇到了什么问题。
3. 第三步是怎么解决的。

## 关键结论

- 结论 1：一句话说清最重要的发现。
- 结论 2：如果别人只记住一个点，希望他记住什么。
- 结论 3：以后再做类似事情时，应该优先检查哪里。

## 可复用清单

- [ ] 把踩坑点整理成 checklist
- [ ] 补充命令、截图或代码片段
- [ ] 给文章加上更准确的标题和摘要

## 结尾

最后可以写：这件事给了你什么启发、下一步准备继续做什么，或者给读者一个延伸阅读入口。
